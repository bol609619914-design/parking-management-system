# 更新日志

## 2026-04-08

### 新增

- 新增用户端真实业务接口：客服工单、电子发票申请、月租续费。
- 新增统一 HTTP 错误工具与 JSON 错误中间件，后端异常返回更加稳定一致。
- 新增用户端状态归一化逻辑，自动汇总优惠券、停车次数和待开发票数量。
- 新增前端常量模块，集中管理角色名称、菜单配置、车位状态与类型文案。

### 优化

- 优化用户端交互，把“联系客服”“开票说明”“月租续费”从详情面板式伪交互补成真实流程。
- 优化车位管理展示，车位状态和车位类型统一改为中文标签。
- 优化登录、注册与找回密码页面的默认值，去掉不必要的预填内容。
- 优化交付文档与项目 README，统一改为正常中文编码并补充 SQLite 单机版说明。

### 修复

- 修复演示种子数据乱码问题，清理用户姓名、告警文案、车牌信息、通知内容等脏数据源。
- 修复用户端电子发票可重复对已开票订单发起申请的问题。
- 修复用户端操作后统计卡片、订单状态和服务提醒不同步的问题。
- 修复后端 API 缺少统一 404 返回的问题。

### 验证

- 已通过 `npm run build` 构建验证。
- 已通过 `npm run db:seed` MySQL 种子数据验证。
- 已验证 MySQL 模式下的用户预约、客服工单、电子发票申请、月租续费、管理员修改计费。
- 已验证权限边界：用户访问后台 OCR 返回 `403`，商户修改计费返回 `403`，重复开票返回 `409`。
- 已验证 SQLite 初始化流程可正常创建单机数据库。

## 2026-04-07

### 新增

- 新增可用的忘记密码流程，支持手机号验证码重置密码。
- 新增商户端计费页只读提示，避免误操作后台配置。
- 新增 Prisma 7 + MySQL 存储层、`prisma.config.ts`、种子脚本与环境变量模板。
- 新增 SQLite 单机模式与便携交付脚本，支持无 MySQL 环境下本地运行。

### 修复

- 修复用户端可越权访问后台 OCR、出入管理、车位控制、计费配置接口的问题。
- 修复商户端可越权修改全局计费规则的问题。
- 修复用户端预约和离场缴费链路中的请求头问题。
- 修复 MySQL 控制台登录链路与便携版启动脚本中的端口、路径和停止逻辑问题。

### 调整

- 管理端保留完整计费配置编辑能力。
- 商户端保留计费规则查看能力，但禁用配置输入与保存按钮。
- 用户端演示数据恢复为“待缴费后离场”状态，便于直接测试预约与缴费。
- 数据访问层支持 JSON、SQLite、MySQL 三种模式切换。

### 验证

- 已完成管理员、商户、用户三端登录验证。
- 已完成管理员 OCR、入场、出场、车位操作、计费保存验证。
- 已完成商户端入场、出场、车位操作验证，并确认计费修改会被正常拦截。
- 已完成用户端预约车位、结束离场缴费、验证码登录、密码重置验证。
