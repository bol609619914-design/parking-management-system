# 更新日志

## 2026-04-07

### 新增

- 新增真实可用的忘记密码流程，支持手机号验证码重置密码。
- 新增商户端计费页只读提示，避免误操作后端配置。
- 新增 Prisma 7 + MySQL 存储层、`prisma.config.ts`、数据库种子脚本与环境变量模板。

### 修复

- 修复用户端可越权访问后台 OCR、入场、出场、车位控制、计费配置接口的问题。
- 修复商户端可越权修改全局计费规则的问题。
- 修复登录页“忘记密码”仅为占位链接、无法完成找回流程的问题。
- 修复用户端演示数据处于已结算状态，导致预约与离场缴费无法直接验证的问题。
- 修复运行日志目录可能进入版本控制的问题，补充 `logs/` 忽略规则。

### 调整

- 管理端保留完整计费配置编辑能力。
- 商户端保留计费规则查看能力，但禁用配置输入与保存按钮。
- 用户端默认演示数据恢复为“待缴费后离场”状态，便于直接测试预约和离场缴费。
- 后端数据访问层改为支持双模式：未配置数据库时使用本地 JSON，配置 `DATABASE_URL` 后切换为 MySQL + Prisma。

### 验证

- 已通过管理员、商户、用户三端登录验证。
- 已通过管理员 OCR、入场、出场、车位操作、计费保存验证。
- 已通过商户入场、出场、车位操作验证，并确认商户修改计费会被正常拦截。
- 已通过用户预约车位、结束离场缴费、短信验证码登录、密码重置验证。
- 已通过 Prisma schema 校验、Prisma Client 生成，以及 JSON 回退模式下的后端健康检查与登录回归验证。
